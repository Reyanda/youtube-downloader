## Objective
Evolve Resource Shrimp from a video+paper downloader into a general resource tool:
pluggable resource types → persistent library → Google Drive → AI assistant over resources.

## Architecture (current)
Zero-dependency Python stdlib HTTP server (`app.py`). stdlib-first; small libs allowed where
stdlib is impractical (decided for Phase C: PDF text extraction).
- `providers.py` — provider registry. `detect()`/`by_name()`/`register()`. VideoProvider
  (yt-dlp, catch-all) + ArticleProvider (DOI/arXiv/PubMed/publisher). Add a type = subclass + register.
- `library.py` — SQLite-backed persistent library. Files under `library/<session>/<id>/`,
  index in `library/library.db`. `persist/list_resources/get/delete`. Session-scoped.
- `app.py` — HTTP + security layer. Anonymous `rs_session` cookie (HttpOnly, SameSite=Lax).
  Endpoints: `/api/download`, `/api/status/<id>` (falls back to library), `/api/stream/<id>`
  (serves from library or temp, no longer deletes), `/api/library` (+`?q=`), `/api/delete`.
  Rate limits split: reads 240/min (polling-safe), downloads 20/min.
- Frontend: `templates/index.html` + `static/app.js` + `static/style.css`. "Library" nav view —
  searchable cards with Download/Delete. Titles HTML-escaped (XSS-safe).
